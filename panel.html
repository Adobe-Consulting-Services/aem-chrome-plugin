AEM Chrome Plug-in could not establish a connection to Sling Log Tracer.

Please follow the setup instructions at:

  • AEM Chrome Plug-in documentation Web site

This error displays when AEM Chrome Plug-in cannot content the AEM instance(s) to collect Sling Log Tracer data. The following are common causes:

  • Apache Sling Log Tracer bundle is not installed, ative, enabled or servlet-enabled on the AEM instance(s).
  • The Sling Log Tracer end-point is blocked (HTTP GET /system/console/tracer should return a 200 status)
  • The Web console user name/password have not been configured in AEM Chrome Plugin > Options
  • The host name used to access AEM in Chrome disallows access to Sling Tracer HTTP end-points.
    Use AEM Chrome Plugin > Options > Host Configurations to map a browser host names to an accessible end-points.
    • This is helpful for using AEM Chrome Plug-in with a multi-AEM backed environment (Example: AEM Publish tier)
  • AEM is configured to use a servlet context, but none has been provided via AEM Chrome Plug-in > Options
  • Another Chrome tab is active (example: AEM Chrome Plug-in > Options) and AEM Chrome Plug-in became mixed up as which is the proper tab to track. Simply reload AEM Chrome Plug-in on your AEM tab.
